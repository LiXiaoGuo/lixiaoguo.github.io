DividerItemDecoration的实现

发表于 2018-03-17 | 分类于 Android | | 阅读次数

谷歌官方有一个DividerItemDecoration控件，专门用来做RecyclerView的分割线，但是扩展性很差，都不能传入高度。遂自己准备实现一个可传入高度和颜色的DividerItemDecoration。

整体思路还是很简单，第一步给每个子View的Rect绘制出间隔来，第二步就是给间隔填充颜色。仅此两步，没了。。。。。。。

我们来具体实现：

1. LinearLayoutManager

LinearLayoutManager比较简单，如果是VERTICAL模式，那么我们只需要画item之间的横线就好了，同理，如果是HORIZONTAL模式，我们只需要画竖线。

给LinearLayoutManager填充颜色也比较简单，可以在循环填充之前判断方向，这样可以有效的避免不必要的计算。

唯一要记住的就是，如果是填充横线的颜色，那么要知道横线的top是view的bottom；如果填充的是竖线，那么线的left是view的right。

2. GridLayoutManager

实现了LinearLayoutManager后觉得GridLayoutManager也不难，遂同理实现了GridLayoutManager，发现效果看起来有点怪，我把间隔设置的很大后发现，最后一列item的宽度明显比前面item的宽度要宽不少。原因就是前面item的宽度被间隔的宽给占据了，而最后一列的右边却是不需要绘制间隔的。

在纸上画了草图后发现：第x列（x从0开始算）的左边偏移量为：$\frac{xy}{n}$ ，右边偏移量为：$\frac{[(n-1)-x]y}{n}$ , y是指间隔的宽度，n表示有几列。

那么我们就可以绘制出GridLayoutManager的间隔了

Netty是什么

Netty是由JBOSS提供的一个java开源框架。Netty提供异步的、事件驱动的网络应用程序框架和工具，用以快速开发高性能、高可靠性的网络服务器和客户端程序。简单的来说，Netty 是一个基于NIO的客户、服务器端编程框架。

之前我们用java提供的原始i/o写了一个简易的长连接，程序很简单，逻辑也很清楚，但是却对服务器的压力很大，因为每一个socket都需要一个Thread来维持。

java 1.4以后引入了一个新的io —— new IO(简称nio)来代替原来的IO。nio的工作方式是基于通道(Channels)和缓冲区(Buffers)的，数据总是从通道读取到缓冲区中，或者从缓冲区写入到通道中，并通过选择器(Selectors)来进行监听管理。这样就可以实现了一个线程去维持多个socket。
